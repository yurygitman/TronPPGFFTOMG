Install the firmware — pick your board, one time

⬡ XIAO ESP32-S3 — one-click flash

Plug in the XIAO, click below, pick its port (e.g. usbmodem…). ~30 s. Already flashed? Skip to step 3.

⚠ This browser can’t flash — open in Chrome, Edge, or Brave on desktop. ⚠ Flashing needs the https:// link (not a local file).

One-click flash is for the XIAO ESP32-S3 only. A different ESP32 (C3 / S2 / classic) shows a “chip doesn’t match” error — use the Arduino · copy & upload path below instead.

Can’t connect? Hold-BOOT trick & tips
  • If the port doesn’t appear or upload fails: hold BOOT, tap RESET, release BOOT, then retry.
  • Close the Arduino IDE / any Serial Monitor first — only one program can hold the port.
  • Use a cable that carries data (some are charge-only). First install offers to erase — that’s fine.
  • No port in the list? Almost always a charge-only cable, or the board needs the hold-BOOT reset above.
▦ Arduino (Uno · Nano · Mega · Leonardo…) — copy & upload

Arduino boards can’t flash from the browser — upload once with the Arduino IDE. After that it connects to the dashboard exactly like the ESP32 (same 6-field stream @ 115200).

  1. Arduino IDE → Tools ▸ Manage Libraries → install “PulseSensor Playground”.
  2. Copy the sketch → new sketch → select your Board & Port → Upload.
  3. Wiring: Signal → A0 · − → GND · + → 5V on 5 V boards (Uno/Nano/Mega) or 3V3 on 3.3 V boards (powering a 3.3 V board’s sensor from 5 V clips the signal).
  4. Back here → step 3 → Connect.

…or download the .ino (identical to the code above). (Same sketch runs on the XIAO too.)

3

See it live

Open the dashboard, click Connect, and pick the same port — you’ll be asked to choose it a second time, and that’s normal (the dashboard connects independently of the flash step). Rest a finger on the sensor: heartbeat shows in a few seconds; breathing detection locks in after ~30–60 s of steady breathing.

Open Live Dashboard →

Tip: in the dashboard, press 🫁 Pace and breathe with the guide (~6/min) to light up the 4-point validator.

BreathRate — cardiorespiratory signal processing, D. John Doyle, MD, PhD. Heartbeat & breathing are derived in your browser from one optical sensor. Live data only — no simulation.

Your privacy: flashing replaces the board’s program (it’ll prompt to erase — expected). Your heartbeat & breathing are processed only in your browser and never uploaded. The only outside calls are the firmware-flash tool’s CDN and an anonymous visit counter.

This is a research/biofeedback tester tool, not a medical device. Web Serial flashing & connection require a Chromium browser on desktop over HTTPS.
